EVAL HARNESS- The eval harness is basically a test suite for an AI system. With normal code, you write unit tests that check "did this function return 42?" With LLMs, the output is non-deterministic — you ask the same question twice and get different wording. So you need a different approach.
What you're testing:
You write test scenarios based on your real training data — situations where you know what a good answer looks like. For example: "User is cutting at 2500 cal, chest volume is 1 set/wk." A good response must flag chest as a gap, must recommend increasing chest volume, must not recommend exceeding MRV, and should cite the Helms deficit guidelines. A bad response would ignore the gap, recommend random exercises, or suggest dangerous volume during an injury.
how it works:
You define a scenario with an input query and a set of evaluation criteria. Then you run the agent, capture the response, and grade it. The grading happens two ways:

LLM-as-judge — You send the agent's response to another LLM with a rubric like "Did this response flag chest as undertrained? Did it provide specific sets/reps? Did it account for the caloric deficit?" The judge scores it 0-1 on each criterion. This is the main evaluation method for AI systems in 2026.
RAG metrics (RAGAS) — These specifically measure your retrieval quality. Faithfulness asks "does the answer actually follow from the retrieved chunks, or did the model hallucinate?" Context relevance asks "did you retrieve the right chunks for this query?" Answer relevance asks "does the response actually address the question?"
Z-score (per muscle group):
Looks at one muscle group at a time. Takes your last ~20 weeks of chest training, calculates your average and spread. Then checks this week against that. If your chest average is 8 sets/wk and this week you did 1, that's way below your personal normal — flag it. Does this independently for every muscle group. Think of it as asking "is this number weird for me?" for each muscle separately.
Isolation Forest (full training profile):
Looks at all muscle groups together as a single snapshot. Each week becomes a row of numbers — chest sets, back sets, biceps sets, quads sets, etc. It learns what your typical training pattern looks like across all muscles simultaneously. Then it asks "does this week's overall pattern look weird?" It catches things like: you normally do 10 sets of chest and 8 sets of triceps together (because they're both push day), but this week you did 0 of both. The z-score would flag each one independently, but the Isolation Forest understands they dropped together, which might mean you skipped push day entirely rather than having two separate gaps.
